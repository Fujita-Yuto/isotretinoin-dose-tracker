# イソトレチノイン累積投与量計算アプリ 要件定義書

## 1. 概要

イソトレチノイン（アキュテイン/ロアキュタン等）を服用中の患者本人が、自分の累積投与量（mg/kg）を計算・記録し、目標値までの進捗を確認できるWebアプリ。

- 対象ユーザー: 服用中の患者本人（日本語話者）
- 対応言語: 日本語のみ
- 利用環境: スマートフォン優先のレスポンシブWeb（PC対応）
- 登録不要・完全無料・広告なし

## 2. 背景知識（計算仕様）

- 累積投与量 (mg/kg) = 服用総量 (mg) ÷ 体重 (kg)
- 服用総量 = Σ（各期間の 1日量 mg × 服用日数）
- 治療終了の目安とされる目標累積量は一般に 120〜150 mg/kg（ユーザーが範囲内で任意に設定可能とする。デフォルト 120 mg/kg）
- 体重は治療中に変動するため、体重の記録も日付付きで複数保持できるようにする。累積量の計算には最新の体重を用いる（シンプルさ優先。将来的には期間ごとの体重を使う計算方式も検討）

## 3. 機能要件

### 3.1 MVP（必須）

1. **体重の記録**
   - 日付 + 体重 (kg) を複数記録できる
   - 最新の体重を累積量計算に使用

2. **服用記録（用量期間ベース）**
   - 「開始日」「1日量 (mg)」を入力すると、その日から次の記録まで（または今日まで）その用量で服用しているとみなす
   - 用量変更・休薬（0mg期間）を履歴として追加できる
   - 記録の編集・削除

3. **累積量の自動計算・表示**
   - 現在の累積投与量 (mg/kg) を大きく表示
   - 服用総量 (mg)、服用日数も表示

4. **進捗表示**
   - 目標値（デフォルト120 mg/kg、ユーザー変更可）に対する進捗バー（％表示）
   - 現在の用量ペースが続いた場合の目標到達予測日

5. **データ保存**
   - ブラウザの localStorage に保存（登録不要）
   - サーバーには一切データを送信しない旨をUI上に明記

6. **免責事項の表示**
   - 「本アプリは計算補助ツールであり、医療アドバイスではありません。服用は必ず医師の指導のもとで行ってください」を初回表示＋フッターに常時表示
   - 催奇形性等の重大な注意がある薬剤であることを踏まえ、自己判断での増減量を推奨しない文言とする

### 3.2 v2以降（任意）

- データのJSON/CSVエクスポート・インポート（機種変更対応、受診時に医師へ提示）
- カレンダー表示・飲み忘れチェック
- 期間ごとの体重を使った、より厳密な累積量計算モード
- 累積量推移のグラフ表示

## 4. 非機能要件

- **プライバシー**: 完全クライアントサイド。アナリティクスを入れる場合もCookieレス（例: Cloudflare Web Analytics）とする
- **パフォーマンス**: 静的サイト。初回ロード軽量、オフラインでも計算可能（PWA化は任意）
- **アクセシビリティ**: スマホ片手操作を想定した大きめの入力UI
- **SEO**: 「イソトレチノイン 累積投与量 計算」等のキーワードでの流入を想定し、計算方法の解説文をトップページに含める

## 5. 技術構成（推奨）

- フレームワーク: Vite + React（または Next.js の静的エクスポート）
- スタイル: Tailwind CSS
- 状態管理: localStorage + React state（外部ライブラリ不要）
- ホスティング: Vercel または Cloudflare Pages（無料枠）
- テスト: 累積量計算ロジックは純関数として切り出し、Vitestでユニットテストを書く（計算の正確性が信頼性の核なので必須）

## 6. 画面構成

1. **ホーム画面**: 現在の累積量・進捗バー・到達予測日・免責フッター
2. **記録画面**: 服用記録（用量履歴）の一覧・追加・編集、体重記録
3. **設定画面**: 目標値の変更、データのリセット（誤操作防止の確認ダイアログ付き）、（v2で）エクスポート/インポート

## 7. 計算ロジックの受け入れ条件（テストケース例）

- 体重60kg、40mg/日を90日間服用 → 総量3600mg、累積量60.0 mg/kg
- 20mg/日を30日 → 40mg/日を60日、体重60kg → 総量 (600 + 2400) = 3000mg、累積量50.0 mg/kg
- 休薬期間（0mg）は日数に含めるが総量には加算しない
- 未来日の記録は入力不可（バリデーション）
